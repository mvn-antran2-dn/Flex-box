What is `use strict`? what are the advantages and disadvantages to using it?

"Use strict " là từ khóa để bật chế độ Strict Mode, tạm dịch là chế độ nghiêm ngặt của JavaScript,
bắt đầu xuất hiện từ phiên bản ECMAScript 5.
Khi khai báo Use strict thì tất cả các dòng code ở phía dưới dòng khai báo Use strict
sẽ được JavaScript quản lý nghiêm ngặt hơn về mặt cú pháp.
Điều này được yêu cầu cực kỳ khắt khe ở các dự án phức tạp để tránh các lỗ hổng tiềm tàng.
Ví dụ: Với Use strict, bạn không thể sử dụng các biến chưa được khai báo.
Từ khóa Use strict có thể đươc sử dụng ở hai vị trí đó là:
Đặt ở đầu file hay đầu code JavaScript có tác dụng cho toàn bộ code JavaScript trong file đó
Đặt ở đầu một hàm có tác dụng cho code trong hàm đó mà thôi.
Lợi Thế:

  • Ngăn chặn sử dụng, và báo lỗi khi người lập trình thực hiện những xử lý được coi là không an toàn, những xử lý mà có thể là ngoài ý muốn.
  • Vô hiệu hoá các tính năng có thể gây nhầm lẫn, hoặc không nên được sử dụng.
  • Ngăn chặn sử dụng một số từ mà có thể sẽ được sử dụng làm keywork trong tương lai.
Bất lợi:
  • Mất đi tính đơn giản, mềm dẻo của ECMAScript thay vào đó là những qui tắc cứng nhắc
  • Khó tiếp cận với những người mới làm quen

What are the differences between `==` and `===`? Write an example for each case (if any)?

Toán tử "==": so sánh trừu tượng (abstract equality), tức là nó thực hiện các chuyển đổi loại cần thiết trước khi so sánh đẳng thức.
Nghĩa là chỉ cần so sánh giá trị của 2 phần tử .Cú pháp so sánh: a == b.
console.log("12 " == 12) ==> true
Toán tử "===": so sánh cân bằng nghiêm ngặt (strict equality), tức là nó sẽ không thực hiện chuyển đổi loại.
Nghĩa là nó so sánh giá trị & kiểu dữ liệu của 2 phần tử.
console.log("12 " === 12) ==> false
